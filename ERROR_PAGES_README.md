# Кастомные страницы ошибок EngAI

Проект теперь включает в себя красивые кастомные страницы ошибок, выполненные в стиле платформы EngAI.

## Созданные файлы

### Шаблоны
- `templates/404.html` - Страница "Страница не найдена" (404 ошибка)
- `templates/500.html` - Страница "Ошибка сервера" (500 ошибка)

### Представления
- `engai/views.py` - Содержит обработчики для кастомных страниц ошибок

### URL конфигурация
- Обновлен `engai/urls.py` с обработчиками ошибок и тестовым URL

## Особенности дизайна

### Страница 404
- **Цветовая схема**: Фиолетовая тема в стиле платформы (#7c4dff)
- **Анимации**: 
  - Плавное появление контейнера (fadeInUp)
  - Подпрыгивающая иконка поиска (bounce)
  - Плавающие фоновые фигуры
  - Анимированные искры
- **Интерактивность**: 
  - Hover эффекты на кнопках
  - Адаптивный дизайн для мобильных устройств
- **Контент**: 
  - Полезная информация о возможностях платформы
  - Кнопки навигации (Главная страница, Назад)

### Страница 500
- **Цветовая схема**: Красно-оранжевая для ошибок сервера (#ff6b6b)
- **Анимации**: Трясущаяся иконка предупреждения (shake)
- **Функциональность**: Кнопки возврата на главную и обновления страницы

## Как использовать

### В режиме разработки (DEBUG=True)
Кастомные страницы ошибок автоматически не работают. Для тестирования:

1. **Тестирование 404 страницы**:
   ```
   http://localhost:8000/404-test/
   ```

2. **Тестирование реальной 404**:
   - Временно установите `DEBUG = False` в settings.py
   - Перейдите на несуществующую страницу
   - Не забудьте вернуть `DEBUG = True`

### В режиме production (DEBUG=False)
Страницы автоматически активируются при:
- 404 ошибке (страница не найдена)
- 500 ошибке (внутренняя ошибка сервера)

## Технические детали

### Обработчики ошибок в urls.py
```python
handler404 = 'engai.views.custom_404_view'
handler500 = 'engai.views.custom_500_view'
```

### Представления в views.py
- `custom_404_view(request, exception)` - для 404 ошибок
- `custom_500_view(request)` - для 500 ошибок
- `page_not_found_view(request)` - для тестирования в режиме DEBUG

## Настройка и кастомизация

### Изменение цветовой схемы
Основные цвета определены в CSS:
- Фиолетовый: `#7c4dff`
- Градиенты: `linear-gradient(135deg, #7c4dff, #6a3edb)`
- Фон: `linear-gradient(135deg, #f8f5ff 0%, #e6d9ff 100%)`

### Добавление новых анимаций
Существующие анимации можно найти в секциях `@keyframes` в CSS.

### Изменение контента
Текст и иконки можно легко изменить в соответствующих HTML файлах.

## Мобильная адаптация
Страницы полностью адаптивны с медиа-запросами для экранов шириной менее 768px.

## Совместимость
- Работает со всеми современными браузерами
- Использует CSS Grid и Flexbox для макетов
- Совместимо с Django 4.2+
